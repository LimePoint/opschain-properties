# CLAUDE.md

## Project Overview

GitHub Actions custom action ("OpsChain Context") that integrates with the OpsChain API to fetch converged configuration properties and environment variables for use in GitHub workflows. Published as `limepoint/opschain-properties`.

This is the **private** repository. The public version is `opschain-properties`, published via a squash-and-publish release process.

## Architecture

- **Runtime:** Node.js 20 (`node20` GitHub Actions runner)
- **Source:** TypeScript in `src/` (`src/index.ts` entry)
- **Bundled output:** `dist/index.js` (built via `@vercel/ncc`, committed to the repo so GitHub Actions can run it without a build step)
- **API client:** `@opschain/api-client` (sibling TypeScript SDK referenced as `file:../opschain-api-client-typescript-sdk` in `package.json`)
- **Action definition:** `action.yml` (`runs.main: dist/index.js`)
- **Release tooling:** `Rakefile` (Ruby/Rake) + `release.sh`

## How It Works

1. Reads inputs: `project`, `environment` (optional), `asset`, `change_id` (optional), `commit_sha` (optional)
2. Reads auth from env vars: `OPSCHAIN_API_URL`, `OPSCHAIN_USERNAME`, `OPSCHAIN_PASSWORD`, `OPSCHAIN_API_AUTH_SCHEME` (default `bearerAuth`)
3. Constructs an `OpsChainClient` via `src/client.ts`; auth options resolved in `src/auth.ts`
4. If `change_id` is provided, fetches change context via `client.changes.get(changeId)` (`src/changes.ts`) and emits `context` output
5. Fetches converged properties via the SDK's auth-aware transport (`client.transport.request("GET", path)`) in `src/properties.ts`. URL is scoped to the environment when `environment` is provided.
6. Strips the `opschain` key from config and `GITHUB_*` keys from env
7. Sets GitHub Action outputs `context`, `config`, `env` (all JSON-stringified)
8. Exports each env entry to the workflow environment via `core.exportVariable`; values starting with `secret-vault://` are masked via `core.setSecret` (`src/env.ts`)

## Key Files

- `src/index.ts` — Entry point; orchestrates inputs, client, fetches, outputs
- `src/client.ts` — `createClient()` and `describeError()` (wraps `OpsChainAPIError`)
- `src/auth.ts` — Resolves `TransportOptions` from env vars; throws `AuthConfigError` on misconfig
- `src/properties.ts` — Fetches converged_properties via auth-aware transport; URL built with optional environment segment
- `src/changes.ts` — Thin wrapper over `client.changes.get()`
- `src/env.ts` — Exports env vars to GitHub workflow; masks `secret-vault://` values
- `action.yml` — GitHub Action metadata; `runs.main: dist/index.js`
- `package.json` — Dependencies (`@actions/core`, `@opschain/api-client` via file: reference) and scripts (`build`, `typecheck`)
- `dist/index.js` — Bundled action (regenerated by `npm run build`; committed)
- `Rakefile` / `release.sh` — Release automation
- `README.md` — Usage documentation and v1→v2 migration guide

## Build & Typecheck

- `npm install` — install deps; resolves the SDK from the sibling directory
- `npm run typecheck` — `tsc --noEmit`
- `npm run build` — `ncc build src/index.ts -o dist --source-map --license licenses.txt`

The `dist/` output **must** be committed; GitHub Actions runs `dist/index.js` directly.

## Release Process

Releases are tagged as `v2` (rolling tag). Use `./release.sh`, which:

1. Runs `npm install && npm run build` (assumes any rebuild has already been committed and pushed to `main`)
2. Deletes the remote `v2` tag
3. Deletes the local `v2` tag
4. `rake private_git_repos:tag_latest` — clones private repo fresh from `origin/main`, force-tags `v2`, force-pushes `main` + tag
5. `rake public_git_repos:publish_squashed_repos` — clones `LimePoint/opschain-properties`, replaces tree with the `v2` snapshot from the private repo, commits as `OpsChain <opschain@limepoint.com>`, force-pushes the public `v2` tag

The Rakefile requires the `rugged` gem and force-pushes `main` and the `v2` tag.

## Git Conventions

- Never include AI attribution (Co-Authored-By, etc.) in commit messages
- Always use the user's git identity for commits (do not override git config); the Rakefile's `OpsChain` identity override only applies to the public squash-publish commit
- Keep commit messages concise and descriptive of the change

## Development Notes

- `node_modules/` is **not** committed (`.gitignore`); `dist/` **is** committed
- No test suite currently (`npm test` is a placeholder)
- Logging uses `core.info` / `core.warning` from `@actions/core`
- Auth is environment-variable driven; action inputs no longer carry API credentials (v2 change; see README "Migrating from v1")
